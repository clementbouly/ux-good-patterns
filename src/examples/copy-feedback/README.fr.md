# Feedback de copie dans le presse-papier

## Description

Quand les utilisateurs cliquent sur un bouton "copier", ils devraient recevoir un feedback visuel immédiat confirmant que le contenu a été copié avec succès dans leur presse-papier.

## Pourquoi c'est important

- **Confirmation** : Les utilisateurs ont besoin de savoir que leur action a réussi avant de passer à autre chose
- **Confiance** : Sans feedback, les utilisateurs peuvent cliquer plusieurs fois ou douter du fonctionnement
- **Accessibilité** : Le feedback visuel aide tous les utilisateurs à comprendre l'état du système
- **Réduction des erreurs** : Les utilisateurs ne colleront pas accidentellement un ancien contenu en pensant que la copie a échoué

## Quand appliquer

- Boutons de copie pour les extraits de code
- Liens de partage ou codes de parrainage
- Clés API ou tokens
- Toute fonctionnalité "copier dans le presse-papier"

## Quand ne pas appliquer

- Quand le contenu copié est immédiatement utilisé dans la même interface (comme un champ de collage)
- Raccourcis clavier (Cmd/Ctrl+C) - les utilisateurs s'attendent à ce qu'ils fonctionnent silencieusement
